Bản thiết kế hệ thống AI nhận diện ngôn ngữ ký hiệu realtime

A. Tổng quan kiến trúc

Hệ thống hoạt động theo một pipeline tuyến tính: webcam → MediaPipe → keypoint extraction → sequence buffer → model inference → smoothing → hiển thị kết quả. Không có microservice, không có cloud — mọi thứ chạy local trên một máy.

Dữ liệu chạy theo hai luồng song song: luồng video (frame BGR từ OpenCV) và luồng keypoints (mảng số thực trích xuất từ MediaPipe). Model chỉ nhìn thấy keypoints, không nhìn thấy pixel — điều này giúp model nhỏ, nhanh, và chạy tốt trên laptop không có GPU.

└── labels.json        # {"Hello": 0, "Thanks": 1, ...}

Mỗi file .npy = 1 sequence = 30 frames × 126 features. Shape: (30, 126).

126 features từ đâu?

  • Left hand: 21 landmarks × 3 (x, y, z) = 63
  • Right hand: 21 landmarks × 3 = 63
  • Tổng: 126 — đủ để phân biệt gesture tay, không quá lớn để train chậm.

Nếu muốn nhận diện gesture toàn thân (như "Hello" vẫy tay kết hợp cơ thể), có thể thêm pose: 33 × 4 = 132, tổng = 258 features.

D. AI Pipeline chi tiết

Input → Keypoint

# mediapipe_detector.py
import mediapipe as mp
import numpy as np

mp_holistic = mp.solutions.holistic

def extract_keypoints(results):
    lh = np.array([[lm.x, lm.y, lm.z] for lm in
                   results.left_hand_landmarks.landmark]).flatten() \
         if results.left_hand_landmarks else np.zeros(63)
    rh = np.array([[lm.x, lm.y, lm.z] for lm in
                   results.right_hand_landmarks.landmark]).flatten() \
         if results.right_hand_landmarks else np.zeros(63)
    return np.concatenate([lh, rh])  # shape: (126,)

Sequence buffer

# sequence_buffer.py
from collections import deque
import numpy as np

class SequenceBuffer:
    def __init__(self, length=30):
        self.buffer = deque(maxlen=length)
        self.length = length

    def add(self, keypoints):          # keypoints shape: (126,)
        self.buffer.append(keypoints)

    def ready(self):
        return len(self.buffer) == self.length

    def get(self):
        return np.array(self.buffer)   # shape: (30, 126)

Model inference + smoothing

# inferencer.py
import tensorflow as tf
import numpy as np
from collections import deque, Counter

model = tf.keras.models.load_model('models/sign_lstm.h5')
smoother = deque(maxlen=10)
labels = {0: "Hello", 1: "Thanks", 2: "Yes", 3: "No", 4: "ILoveYou"} # Load từ JSON

def predict(sequence):
    # sequence shape: (30, 126) → add batch dim → (1, 30, 126)
    probs = model.predict(sequence[np.newaxis], verbose=0)[0]
    confidence = np.max(probs)
    label_idx = np.argmax(probs)

    if confidence > 0.7:
        smoother.append(label_idx)
    
    # Majority vote trên 10 predictions gần nhất
    if len(smoother) == 10:
        final = Counter(smoother).most_common(1)[0][0]
        return labels[final], confidence
    return None, 0.0

E. Model Architecture

So sánh các mô hình

LSTM (Khuyên dùng)

  • Dễ train, Realtime OK, Ổn định.
  • Xử lý chuỗi thời gian tốt, nhiều tutorial, Keras fit trong ~5 phút.
  • Lý tưởng cho deadline 3 tuần.
  • Inference ~2ms/frame

GRU

  • Nhẹ hơn LSTM, Nhanh hơn.
  • Kém hơn 1-2% accuracy so với LSTM.
  • Ít tham số hơn LSTM ~30%, thường đủ tốt. Upgrade nhỏ nếu LSTM chạy chậm.
  • Inference ~1.5ms/frame

CNN-LSTM

  • Accuracy cao.
  • Phức tạp hơn, Train lâu hơn.
  • CNN extract spatial feature từ keypoints, LSTM xử lý temporal. Tốt nhưng over-engineering cho 3 tuần.
  • Inference ~5ms/frame

Transformer

  • SOTA accuracy.
  • Train cần GPU, Debug khó.
  • Mạnh nhất nhưng cần nhiều data, nhiều RAM, và hiểu biết sâu. Để tuần 3 nếu có thời gian.
  • Inference ~8ms/frame

MLP + RF

  • Train cực nhanh.
  • Không có temporal, Accuracy thấp.
  • Bỏ qua thông tin chuỗi thời gian. Dùng để baseline so sánh, không dùng production.
  • Inference ~0.1ms/frame

Kiến trúc LSTM khuyến nghị

# model.py
import tensorflow as tf

def build_model(num_classes=5, seq_len=30, features=126):
    model = tf.keras.Sequential([
        tf.keras.layers.LSTM(64, return_sequences=True,
                             input_shape=(seq_len, features)),
        tf.keras.layers.Dropout(0.2),
        tf.keras.layers.LSTM(128, return_sequences=True),
        tf.keras.layers.Dropout(0.2),
        tf.keras.layers.LSTM(64),
        tf.keras.layers.Dense(64, activation='relu'),
        tf.keras.layers.Dense(num_classes, activation='softmax')
    ])
    model.compile(optimizer='adam',
                  loss='categorical_crossentropy',
                  metrics=['accuracy'])
    return model

# Tổng params: ~130k — rất nhẹ, train ~5 phút trên CPU

I. Realtime optimization — những điều quan trọng

  • FPS target: 20fps trở lên trên laptop i5 không GPU.
  • Không resize lớn: dùng 640×480 là đủ, MediaPipe không cần 1080p.
  • Skip MediaPipe detection khi không cần: nếu frame trước không có tay, vẫn inference nhưng ghi zeros.
  • Chạy model mỗi frame, không cần throttle: LSTM 130k params inference ~2ms, không đáng throttle.
  • Dùng deque(maxlen=30): thay list để tránh re-allocate memory mỗi frame.
  • Tắt verbose inference: model.predict(x, verbose=0) tránh print log làm chậm terminal.
  • Profile: bằng time.perf_counter() để biết bottleneck ở đâu trước khi optimize.

J. Hướng nâng cấp sau 3 tuần

Sau khi demo xong, nếu muốn đưa vào CV project hoặc tiếp tục nghiên cứu:

  • Data: thu thêm data (100 sequences/gesture thay vì 30), thêm người thu để mô hình generalize tốt hơn.
  • Model: thử Transformer nhỏ (2-4 heads, 2 layers) — có thể beat LSTM 2-3% accuracy nếu có đủ data.
  • Gesture mới: pipeline đã thiết kế để dễ mở rộng — chỉ cần chạy collect_data.py với label mới, retrain là xong.
  • Vietnamese Sign Language (VSL): dataset VSL chưa nhiều, nhưng pipeline này hoàn toàn apply được — chỉ cần thu data theo bộ từ VSL.
  • Mobile deployment: convert .h5 sang TFLite, chạy trên Android/iOS với MediaPipe Mobile SDK. Inference time < 5ms trên điện thoại tầm trung.
  • Edge AI: chạy trên Raspberry Pi 4 + camera module, demo embedded AI — rất ấn tượng với nhà tuyển dụng.
Tóm lại cho nhóm: Tuần 1 tập trung vào data sạch và MediaPipe stable. Tuần 2 LSTM 90%+ là milestone bắt buộc. Tuần 3 làm đẹp và demo. Đừng tốn thời gian Transformer hay CNN-LSTM nếu LSTM chưa đạt 90% — tăng data trước, đổi model sau.
