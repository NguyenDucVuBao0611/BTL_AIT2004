Hệ thống Nhận dạng Ngôn ngữ Ký hiệu

A. Tổng quan kiến ​​trúc

Hệ thống bao gồm 6 lớp độc lập: người dùng truy cập trình duyệt (webcam), qua proxy Nginx đến API cuối cùng, điều phối phụ trợ giữa Redis Queue, PostgreSQL Sớm chuyển đến Dịch vụ AI. AI Service xử lý khung theo quy trình 5 bước rồi trả kết quả JSON về Backend → TTS phát âm và hiển thị văn bản trên Frontend.

Luồng dữ liệu chính:
Khung hình webcam (JPEG/30fps) → WebSocket → Backend → Hàng đợi Redis → Dịch vụ AI (MediaPipe + LSTM) → Kết quả JSON → Backend → WebSocket → Frontend (văn bản + âm thanh)

C. Cơ sở dữ liệu thiết kế

-- Bảng từ vựng ký hiệu
CREATE TABLE vocabulary (
  id         SERIAL PRIMARY KEY,
  sign_label VARCHAR(100) NOT NULL UNIQUE,  -- "hello", "thank_you"...
  description TEXT,
  video_demo_url VARCHAR(255),
  language   VARCHAR(10) DEFAULT 'vn',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Bảng lịch sử dịch
CREATE TABLE translation_history (
  id           SERIAL PRIMARY KEY,
  session_id   UUID NOT NULL,
  sign_label   VARCHAR(100) REFERENCES vocabulary(sign_label),
  confidence   FLOAT,           -- 0.0 – 1.0
  raw_text     TEXT,
  tts_audio_url VARCHAR(255),   -- optional, cache audio
  translated_at TIMESTAMP DEFAULT NOW()
);

D. Pipeline AI — Chi tiết

Tại đây bạn có thể tìm thấy MediaPipe + LSTM và 3D-CNN:

  • MediaPipe (của Google, đã được huấn luyện trước) có thể xử lý đồng thời 21 điểm đặc trưng và 33 điểm đặc trưng tư thế (~5ms/khung hình) — bộ trích xuất đặc trưng hiện đã có sẵn.
  • LSTM nhẹ nhàng (~500KB), huấn luyện nhanh với tập dữ liệu nhỏ (~1000 mẫu/lớp), phù hợp thời gian 3 tuần.
  • Tổng thời gian suy luận: ~30–50ms/khung hình trên CPU, trong thời gian thực.
Frame JPEG (640×480) 
 ↓ OpenCV resize → 640×480 normalize 
 ↓ MediaPipe Holistic → 21 hand + 33 pose landmarks = 63 keypoints × 3 (x,y,z) = 189 float/frame 
 ↓ Buffer 30 frames liên tiếp → tensor shape [30, 189] 
 ↓ LSTM(128) → LSTM(64) → Dense(num_classes) → Softmax 
 ↓ Sliding window (5 frame majority vote) + threshold ≥ 0.85 
 ↓ Output: {"label": "xin_chào", "confidence": 0.93}

Hậu xử lý (Làm mịn): Dùng cửa sổ trượt 5 khung — chỉ công nhận kết quả khi ≥3/5 khung liên tiếp đồng ý cùng 1 nhãn, tránh giật/nhiễu.

E. Luồng dữ liệu thời gian thực (WebSocket)

  1. Frontend: getUserMedia() → canvas.captureStream(15fps)
  2. Frontend: encode frame → base64 JPEG → send qua WebSocket
  3. Backend: nhận frame → push vào Redis List (LPUSH frame_queue)
  4. AI Service: BLPOP frame_queue → xử lý pipeline → publish kết quả Redis channel
  5. Backend: subscribe Redis channel → nhận kết quả → gửi về Frontend qua WebSocket
  6. Frontend: nhận JSON {label, confidence} → hiển thị text + gọi TTS

I. Điểm cộng báo cáo

  • Tăng cường dữ liệu cho keypoint (Data Augmentation):
    • Xoay ngẫu nhiên ±15° trên các điểm chính.
    • Biến dạng thời gian (kéo giãn/nén 30 khung hình).
    • Mirror flip (đối xứng tay trái/phải) → Tăng tập dữ liệu 5–10× chỉ bằng numpy.
  • Tối ưu tốc độ suy luận:
    • Chuyển mô hình sang TensorFlow Lite → giảm từ ~80ms xuống ~20ms/frame.
    • Hoặc dùng ONNX Runtime xuất từ ​​Keras.
    • Báo cáo benchmark: FPS trước/sau → số liệu cụ thể rất thuyết phục.
  • Bonus nhỏ nhưng hay:
    • Thanh tin cậy (Confidence bar) hiển thị thời gian thực trên UI.
    • Cửa sổ trượt đa lớp (hiển thị top 3 dự đoán).
    • Log độ trễ của WebSocket → chứng minh thời gian thực < 100ms.
  • Gợi ý tập dữ liệu để bắt đầu nhanh: Sử dụng WLASL hoặc tự quay 5–10 ký hiệu tiếng Việt cơ bản (xin chào, cảm ơn, có, không, tạm biệt). Với MediaPipe + LSTM, 200 mẫu/lớp có thể đạt độ chính xác >85%.
